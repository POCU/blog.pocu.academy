부정행위

윤아 2020-12-20
  • 가이드
  • 운영

안녕하세요, POCU 아카데미 수강생 여러분. 윤아입니다!

오늘은 부정행위에 대해 알아보겠습니다. 어떤 건 부정행위에 해당하고, 어떤 건 부정행위에 해당하지 않는지 정확히 알아야 좀 더 활발하게 POCU에서 활동할 수 있겠죠? 부정행위인 줄 모르고 행동했다가 제재를 받게 되거나, 반대로 부정행위인 줄 알고 정당한 권리를 행사하지 못하는 일이 발생하면 억울하니까요!

부정행위의 기본 원칙

부정행위가 무엇인지 알기 전에 먼저 POCU 아카데미에 적용되는 부정행위의 기본 원칙을 간략하게 설명해드릴게요. 이렇게 부정행위를 꼼꼼하게 체크하는 이유는 원활한 교육 환경 조성을 위해서랍니다. 교육의 기회는 공평해야 하기 때문에 혹시 누군가의 부정한 행동으로 나의 교육권이 침해되거나 불이익을 얻게 되면 안 되죠!

  • POCU 아카데미는 교육 과정에서 비윤리적인 학습 행위가 의심되는 경우 조사를 할 수 있으며, 그에 따라 징계를 내릴 수 있습니다. 징계의 최대 범위는 정학!

  • 또한 POCU 아카데미는 비윤리적인 행위를 발견한 즉시 개입을 할 것입니다.

  • 아래의 부정행위 예시들을 몰랐다고 해도 그대로 처벌받을 수 있음을 꼭 알아두세요.

부정행위

POCU 아카데미에서 배우는 모든 학생들은 정직함을 유지해야 합니다. 정직함은 배움에 있어서 기초적인 약속이죠. 저희는 학생 여러분들에게 자신의 이름으로 제출하는 모든 작품은 본인이 독립적으로 만든 작품이어야 하고, 시험 과정에서 규정을 준수하며, 보안 및 저작권을 보호하는 등의 정직한 행동을 기대합니다. 반대로 학습윤리에 어긋나는 행동을 한다면 적절한 조치가 이루어져야겠죠.

아래 예시들은 편의를 위해 제공하는 것으로, 여기에 해당하지 않는 행위도 부정행위로 성립할 수 있답니다.

A. 실습/과제 중 부정행위

1. 내가 직접 작성하지 않은 코드를 제출하는 행위

출처를 불문하고 내가 직접 작성하지 않은 코드를 제출하는 것은 부정행위로 처리된답니다. 인터넷 검색을 통해 찾은 다른 코드를 참고할 수는 있으나 그것은 이해를 돕기 위한 용도로만 쓰세요. 실제 제출하는 코드를 작성할 때는 그 참고자료를 덮고 스스로 코드를 작성하시는 게 최고!

단, 이 규칙에도 예외가 있답니다. 사실 하나뿐인데, POCU 아카데미에서 제공한 코드는 사용해도 상관없다고 하네요~

2. 내 코드를 타 학생에게 제공하는 행위

학생들 사이에서 제출 마감 전에 실습/과제의 코드를 주고받는 일체의 행위를 금지합니다. 코드의 일부라도 공유하면 부정행위에 해당하니 조심! 가끔 슬랙방에서 순수하게 질문 목적으로 코드를 공유하는 경우가 있는데, 모르고 하셨다고 해도 부정행위에 해당하니 주의해주세요.

예시

학생: 이 부분이 막혀서 그런데 코드 좀 봐주시겠어요?

// 코드 공유
int main(void)
{
    printf("hello world!\n");
    return 0;
}

3. 내 코드를 불특정 다수에게 공개하는 행위

내 코드를 불특정 다수에게 공개하는 것(예: git 리포를 공개, 블로그에 포스팅)도 부정행위가 될 수 있데요! ㅠ_ㅠ 다른 학생이 그 코드를 복사해 제출할 수 있거든요. 이럴 경우 두 학생 모두 부정행위로 처리되니 특히 학기 진행 중에는 조심하세요~

B. 시험 중 부정행위

1. 시험 중 컨닝 페이퍼나 기타 참고자료 사용하는 행위

이건 말할 필요도 없겠죠? 학창 시절에 컨닝하다 걸리면 어떻게 되는지 아시죠… -_- (장난치다 걸리면 피보는 거 안 배웠냐.. feat. 아귀)
대신 사전에 허락을 받은 참고자료를 사용하는 것은 가능하답니다. 예를 들어 COMP1000:소프트웨어 공학용 수학의 경우 시험 중 두 장의 노트 사용을 허락하고 있습니다. (수학 문제를 머리로 풀 수는 없기 때문에..)

2. 시험 중 문제에 대한 답을 주고받기 위해 다른 학생과 소통하는 행위

POCU에서 시험을 칠 때는 웹캠이 필수입니다. 시험 치기 전 본인 인증을 철저하게 하고, 심지어 시험 환경의 360도를 동영상으로 찍게 해 혹시나 모를 부정행위를 방지합니다. 그리고 시험을 칠 때 크롬 확장 프로그램을 필수로 설치해야 시험을 칠 수 있습니다. 이 프로그램은 학생의 시험 화면을 녹화하고 시험을 보는 창 외에 다른 프로그램이 실행되고 있지는 않은지 감시합니다.

3. 시험에서 다른 학생으로 속이는 행위

수능 시험에서도 대리 시험을 치다가 걸리면 5년 이하의 징역 또는 1000만 원 이하의 벌금에 처하도록 되어 있죠(ㅎㄷㄷ). 그만큼 대리 시험은 비윤리적인 행위입니다. 대리 시험을 막기 위해 시험 전 본인의 신분증 사진을 찍고 시험을 치도록 안내합니다.

이 외에 타인이 위에 열거된 행동을 할 수 있도록 만드는 일체의 행위를 금합니다!

부정행위가 아닌 것

너무 겁을 많이 드려서 아무 질문도 토론도 안 하게 되는 건 아닌지 걱정이군요. 허허;; 그렇다면 부정행위일까 아닐까 헷갈리지만 부정행위에 속하지 않는 행동은 어떤 것이 있을까요?

1. 의사 코드로 과제/실습 공유

의사(비교할 의, 비슷할 사. Pseudo: 가짜의- ) 코드는 컴퓨터 프로그램이나 알고리듬이 수행해야 할 내용을 우리가 사용하는 언어로 간략히 서술해 놓은 것을 말합니다.

실습/과제의 코드를 직접 공유하며 질문하는 것은 부정행위에 해당하지만, 의사 코드로 코드를 설명하거나 말로 코드의 흐름을 설명하는 것은 오히려 권장하고 있습니다.

예시

학생: 문자열에서 ‘a’를 찾는 코드를 짜려는데 무한루프가 나네요 ㅠㅠ 흐름은 이렇습니다.

// 의사 코드 공유
while (문자가 '\0'이 아닐 때) {
    if 문자가 'a'라면 break;
}

답변: while문 끝에 포인터를 증가시켜주는 코드를 작성하셔야 해요.

2. 이미 공개되어 있는 정보에 관한 토론

웹 검색을 통해 누구나 알 수 있는 코드 혹은 정보를 공유하는 것은 부정행위가 아닙니다.

예시

학생: 논리 연산자는 short circuit이 적용되는군요!

int a = 0;
int b = 0;
if (++a || b) {
    printf("short circuit!");
}

3. 시험 혹은 실습/과제 기한이 지난 후 답안/코드 공유

시험 기간 중에 본인이 시험을 다쳤다고 시험 문제에 대한 언급을 하면 부정행위에 해당합니다. 하지만 시험 기간이 지나고 나면 얼마든 시험에 관해 토론하셔도 괜찮습니다.

또한 실습/과제 제출 기한이 지나고 코드 공유를 하는 것은 적극 권장합니다. 다른 학생분들의 코드를 보며 배우기도 하고 내 코드에 대한 피드백을 받으며 서로 좋은 영향을 주고받을 수 있기 때문이죠 ^-^

마치며

이상 부정행위에 대해 자세히 알아봤습니다. POCU는 절대 부정행위를 용인하지 않는다는 점! 그리고 발견 즉시 해당 행위에 대한 징계를 내린다는 점! 꼭 명심하세요. 다른 학우들의 교육권과 더 쾌적한 교육 환경을 위해 다 함께 노력해요! (땀은 배신하지 않는다능..)
